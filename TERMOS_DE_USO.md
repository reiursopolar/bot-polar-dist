## 3. Telemetria e dados de diagnóstico

Para melhorar a qualidade do serviço e prestar suporte técnico, o Polar Bot recolhe e envia automaticamente para os servidores do criador os seguintes dados de diagnóstico:

### 3.1 O que é recolhido

| Dado | Descrição |
|------|-----------|
| **Estatísticas de comandos** | Nome dos comandos utilizados e o número de vezes que foram usados. Não inclui argumentos, mensagens ou conteúdo. |
| **Relatórios de erros** | Tipo de erro, mensagem de erro e traceback técnico (máximo 4 linhas) quando ocorre uma falha no bot. |
| **Estado do bot** | Versão do software instalada, data/hora de arranque e data do último contacto com os servidores. |

### 3.2 O que NÃO é recolhido

- Conteúdo de mensagens (texto, imagens, áudio, vídeo, documentos)
- Números de telefone dos utilizadores do teu bot
- Nomes ou membros de grupos
- Chaves de API que tens configuradas (Groq, Mercado Pago, etc.)
- Informações de pagamento

### 3.3 Como é usado

Os dados recolhidos são usados exclusivamente para:
- Identificar e corrigir bugs e falhas técnicas
- Entender que funcionalidades são mais utilizadas (para as melhorar)
- Verificar a integridade das licenças em uso

### 3.4 Onde é armazenado

Os dados são armazenados nos servidores Cloudflare Workers, associados à tua chave de licença. O criador do Polar Bot pode consultar estes dados no painel de gestão.
